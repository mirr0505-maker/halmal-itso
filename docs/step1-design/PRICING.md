## 0. 화폐 기준

### 0.1 기본 환산

| 단위 | 볼 | 원화 |
|------|:----:|:------:|
| 1볼 | 1 | **100원** |
| 10볼 | 10 | 1,000원 |
| 50볼 | 50 | 5,000원 |
| 100볼 | 100 | 10,000원 |
| 300볼 | 300 | 30,000원 |
| 500볼 | 500 | 50,000원 |
| 1,000볼 | 1,000 | 100,000원 |

### 0.2 기준 근거

- 땡스볼 1회가 **"100원어치 감사 표현"**으로 직관적
- 1,000원 후원 = 10볼 (간결한 단위)
- 유배 1차 속죄금 10볼 = 1,000원 → "경미한 벌금" 뉘앙스와 일치
- **비과세 한도** (건당 5만원) = 500볼 → 송금 상한 재조정 근거

### 0.3 문서 표기 원칙

- 모든 볼 금액에 **원화 병기** 권장 (단, 통합표에서는 생략 가능)
- 예: `100볼(10,000원)`, `30볼(3,000원)`
- 가격 변경 시 **볼 기준**으로 결정 (환율 개편 시 원화만 재계산)

## 1. 땡스볼 (후원·라이브)

### 1.1 송금 한도

| 항목 | 현재 실측 | 원화 | 비고 |
|------|:--------:|:----:|------|
| 1회 송금 최소 | 1볼 | 100원 | `thanksball.js` `amount >= 1` |
| **1회 송금 최대** | **10,000볼** | **1,000,000원** | ⚠️ `MAX_AMOUNT_PER_TX=10000` 🔴 과다 |
| 연속 송금 최소 간격 | 3초 | — | `MIN_INTERVAL_MS=3000` |

### 1.2 🔴 권장 변경: 1회 송금 상한 재조정

**현재 문제**: 10,000볼(100만원)은 너무 큼

- 세무상 **"증여세 비과세 한도"** = 건당 5만원
- 5만원 이상 송금 시 원천징수 대상 우려
- 자금세탁 리스크

**권장안**:

| 제안 | 한도 | 원화 | 근거 |
|------|:----:|:----:|------|
| 🟢 보수적 | 500볼 | 50,000원 | 비과세 한도 정확 일치 |
| 🟡 중도 | 1,000볼 | 100,000원 | 여유 + 관리 가능 |
| 🔴 현재 유지 | 10,000볼 | 1,000,000원 | 리스크 큼 |

**권장**: **500볼 (50,000원)** — PG 연동 시점 적용. 현재는 MVP 단계라 일단 유지, `ANTI_ABUSE.md` TODO 기록.

## 4. 잉크병 (유료 회차 연재)

### 4.1 플랫폼 수수료

**출처**: `functions/inkwell.js` `PLATFORM_FEE_RATE = 0.11`

| 항목 | 값 |
|------|:---:|
| 플랫폼 수수료 | **11%** |
| 작가 수령 | **89%** |

### 4.2 가격 범위

**출처**: `types.ts` `Series.defaultPrice`

- 최소: 1볼 (100원)
- 권장 (웹소설 1화): 2~5볼 (200원~500원)
- 최대: 작가 자율 (상한 없음, 단 상식적 범위)

### 4.3 구체 예시

**회차 5볼(500원) 판매 시**:

| 항목 | 볼 | 원화 |
|------|:---:|:----:|
| 구매자 결제 | 5볼 | 500원 |
| 플랫폼 수수료 (11%) | 0볼 (Math.floor(5×0.11)=0) | 0원 |
| 작가 수령 | 5볼 | 500원 |

*주의: 저가격대(1~9볼)에서는 `Math.floor`로 수수료가 0이 됨 → 고가격대에 수수료 집중*

**회차 50볼(5,000원) 판매 시**:

| 항목 | 볼 | 원화 |
|------|:---:|:----:|
| 구매자 결제 | 50볼 | 5,000원 |
| 플랫폼 수수료 (11%) | 5볼 | 500원 |
| 작가 수령 | 45볼 | 4,500원 |

### 4.4 작품 구독 (알림만, 과금 없음)

- 구독 자체는 무료 (`series_subscriptions` 문서 생성)
- 새 회차 발행 시 `new_episode` 알림 발송
- 회차별 개별 결제는 별도

## 8. 닉네임 변경 수수료 (신규 — v2 확정)

### 8.1 정책 요약

**참조**: `GLOVE_SYSTEM_REDESIGN_v2.md §5.3.5`

| 항목 | 값 |
|------|:---:|
| **가입 후 변경** | 금지 |
| **평생 1회 유료 변경** | 허용 |
| **수수료** | **100볼 (10,000원)** |
| **수익 배분** | 플랫폼 **100%** (`platform_revenue/nickname_change`에 누적) |
| **이전 닉네임** | 영구 예약 (재사용 불가) |
| **공개 프로필** | 변경 이력 영구 표시 |

### 8.2 수수료 근거

**왜 100볼 (10,000원)?**
- 5,000원(50볼)은 너무 가벼움 → 신중함 유도 부족
- 20,000원(200볼)은 진짜 필요한 사람에게도 부담
- **10,000원**은 치킨 반 마리 값 = "의미 있지만 감당 가능" 범위
- 카카오톡(무료)·인스타(무료)와 차별화 → "신뢰성 중심 플랫폼" 포지셔닝

### 8.3 예외 케이스

- **신변 위협 등 특수 상황**: 관리자가 수동 변경 (`sanction_log` 기록), `nicknameChangeCount` 미증가
- **CF 경유 변경**: `changeNickname` 신규 CF (Rules는 CF 전용으로 잠금)

## 10. 세금·정산 파라미터

### 10.3 원천징수 계산

```typescript
// utils.ts 기존 함수
export const calculateWithholdingTax = (
  grossAmount: number,
  incomeType: 'business' | 'other'
): { taxAmount: number; netAmount: number; taxRate: number } => {
  const taxRate = incomeType === 'business' ? 0.033 : 0.088;
  const taxAmount = Math.floor(grossAmount * taxRate);
  return { taxAmount, netAmount: grossAmount - taxAmount, taxRate };
};
```

- 원천세 (3.3%): 3,300원
