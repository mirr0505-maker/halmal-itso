## 1. 컨셉 & 네이밍

### 의미
숲속에서 길을 찾기 위해 흘려둔 빵부스러기처럼, 독자들을 맛있는 과자집(오리지널 긴 글)으로 유도하는 매력적인 '조각'들. **"글의 하이라이트만 뽑아서 꼬셔온다"**는 기획 의도를 이솝우화/동화 세계관으로 은유.

### 핵심 목표
- **Hook → Conversion**: 짧은 이미지로 시선을 사로잡고, 결국 오리지널 긴 글로 유도
- **유튜브 쇼츠와의 차별점**: 숏폼 자체로 소비가 끝나지 않고, 반드시 원본글로 흘러가는 구조

## 2. 한컷 vs 네컷 — 병행 운영

**결론**: 1~4장까지 자유롭게 선택 가능한 캐러셀 형태. 한컷도 의미가 있음.

| 형태 | 비유 | 강점 |
|------|------|------|
| **1컷** | 영화 포스터 | 강렬한 시선 집중 (어그로) |
| **4컷** | 영화 예고편 | 기-승-전-결 스토리텔링 |

### 4컷의 황금 구조 (기-승-전-결)
1. **1컷**: 시선 집중 (어그로)
2. **2컷**: 상황 설명 (공감)
3. **3컷**: 위기 또는 반전 (호기심 극대화)
4. **4컷**: **절단신공** (결말은 안 보여주고 "본문에서 확인하세요" + 링크 버튼)

> 인스타 카드뉴스, 인생네컷 열풍이 증명하듯 사람들은 스와이프로 3~4장을 소비하는 것에 완벽 적응되어 있음.

## 3. 데이터 모델 변경 (하위호환 유지)

### 3.1 `Post` 인터페이스 확장 (`src/types.ts`)

```typescript
interface Post {
  // ...기존 필드 유지
  imageUrl?: string | null;        // 기존 — 1컷 호환용 + 썸네일 fallback
  imageUrls?: string[];            // 🆕 신규 — 1~4장 배열 (네컷 캐러셀)
}
```

**원칙**:
- 기존 한컷 글(`imageUrl`만 있는 글)은 **마이그레이션 불필요**
- 신규 작성 글은 `imageUrls` 배열에 1~4장 저장 + `imageUrl`에는 첫 장 복사 (썸네일 fallback)
- 렌더링 시: `imageUrls?.length ? imageUrls : (imageUrl ? [imageUrl] : [])`
- 컷 수 = `imageUrls?.length ?? (imageUrl ? 1 : 0)`

### 3.2 Firestore 영향
- **Rules 변경 불필요** — 기존 posts write 권한 안에서 동작
- **인덱스 변경 불필요**

## 5. UI 상세 설계

### 5.1 작성 폼 (`CreateOneCutBox.tsx`)

#### 업로드 슬롯
- **1~4슬롯** 표시 (드래그·클릭 업로드)
- 16:9 비율 유지 (`aspect-[16/9]`)
- 슬롯 1은 필수, 2~4는 선택
- 슬롯 추가 버튼: `+ 다음 컷 추가` (최대 4컷까지)
- 각 슬롯 우상단에 `×` 삭제 버튼
- 슬롯 순서 변경 (드래그 앤 드롭) — **v1에서는 보류**, v1.1 추후 추가

#### 가이드 박스 (인라인 안내)
```
💡 빵부스러기 잘 뿌리는 법

1컷: 시선 집중 (어그로)
2컷: 상황 설명 (공감)
3컷: 위기 또는 반전 (호기심 극대화)
4컷: 절단신공! (결말은 X, 원본글 링크는 O)

👉 마지막 컷에서 말을 끊으세요. 요약본이 아니라 '예고편'을 만드세요!
```

#### 원본글 링크 (linkedPostId)
- **1컷·4컷 모두 선택사항** (자율성 우선)
- 기존 `LinkSearchModal` 재사용
- 외부 링크(`linkUrl`)도 그대로 허용

#### 미리보기
- 캐러셀 형태 (좌/우 화살표 + 인디케이터)
- 마지막 장에는 미리보기로 CTA 오버레이 표시 (linkedPostId or linkUrl 있을 때)

### 5.2 상세 뷰 (`OneCutDetailView.tsx`)

#### 캐러셀
- 좌/우 화살표 버튼 + 키보드 ←/→ 지원
- 하단 인디케이터 (점 N개, 현재 위치 강조)
- 모바일: 스와이프 제스처
- 16:9 비율 유지

#### 마지막 장 CTA 오버레이 ⭐ 핵심
- **위치**: 마지막 컷 이미지 위 **하단 중앙 오버레이**
- **노출 조건**: `linkedPostId` 또는 `linkUrl`이 있을 때만
- **디자인**:
  ```
  [🔗 숨겨진 자세한 이야기 보러가기]
  ```
  > 네컷은 절단신공뿐 아니라 **요약본** 용도로도 쓰일 수 있으므로 "뒷이야기"가 아닌 "자세한 이야기"로 일반화 (v1.1)
  - 큼직한 버튼 (`px-6 py-3 rounded-full`)
  - 어두운 반투명 배경 + 흰 글씨 + 그림자
  - 살짝 떠오르는 애니메이션 (호버 시 scale-105)
- **클릭 동작**:
  - `linkedPostId` 우선 → 해당 게시글로 이동
  - 없으면 `linkUrl` → 외부 링크 새 탭

#### 1컷일 때
- 캐러셀 UI 숨김 (단순 이미지 1장)
- CTA 오버레이는 동일하게 표시 (linkedPostId 있을 때)

### 5.3 리스트 카드 (`OneCutList.tsx`, `PostCard.tsx`)

#### 컷 수 배지
- 위치: 썸네일 우상단
- 디자인: `🍞 1/4` (배경 검정 반투명, 흰 글씨, `rounded-full px-2 py-0.5 text-xs`)
- 1컷일 때도 `🍞 1/1` 표시 (통합 일관성)
- 썸네일은 `imageUrls[0] ?? imageUrl` 사용
